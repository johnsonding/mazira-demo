Simplify Client-Side Script Management for your Next Web Project

by Uday Verma
2013/07/07

TL;DR

Check out the fetch.js project hosted at Mazira's GitHub portal for a nice, lean way for managing your client side scripts.

npm install -g fetchjs
fetchjs init
fetchjs search lodash
fetchjs get lodash.js > public/js/lodash.js
fetchjs get jquery async lodash.js > public/js/vendor.js

Introduction

We often code a lot of quick prototypes at Mazira. Every time I start coding a new front-end I despise the process where I have to go out on the Internet and figure out CDNs or download locations for all the scripts I may need. I also prefer to keep my scripts local to avoid surprises when things change and to be able to work when I am at places where my Internet connectivity is not as great (coding outside during the summer is awesome at Mazira's main office in Iowa City).

There are packages out there that take care of managing these client side libraries for you. Most of them have either been too elaborate for my attention span or have just too much boilerplate, which makes using them not so lean and mean.

We, therefore, came up with a way to get these scripts on our dev machines quickly and without much hassle.

Introducing fetch.js

fetch.js makes it trivial to download and setup your client side scripts. Its a simple command (written on top of Node.js) which you use from the command line. It relies on the awesome cdnjs's github repo.

How to use?

Setting up fetch.js is simple, just run:

sudo npm install -g fetchjs

You should now have fetchjs command available. The first thing you need to do right after install is to init the fetchjs repo:

fetchjs init

This command basically clones the cdnjs repo to your ~/.fetchjs directory. Bringing your repo to the latest version is simple as well:

fetchjs update

Once you have the repo setup, you are ready to starting fetching scripts for your packages of interest.

To search for a package you can issue the search command with a pattern:

fetchjs search lodash

This command will list all the packages available in the cdnjs repo which match "lodash". Getting a package is simple as well.

fetchjs get lodash.js

This will dump everything from lodash.js to your console. You can redirect to a file to capture results:

fetchjs get lodash.js > public/js/lodash.js

You can get multiple packages and dump them all into a single vendor scripts file.

fetchjs get jquery jqueryui async lodash > public/js/vendor.js

Now you have all your dependency packages in one javascript file which you can link to from your webpages.

Future work

Not a long list of things we can do with this, but some things that'd be nice to have:

  • Integration with npm to automatically install dependent client side packages.
  • Ability to switch between minified and non-minified versions.
